# 接口规范

DYBridge.dispatch(clazz,method,json,funtion(code,msg,data){

})

所有的接口都是如上形式：

  • DYBridge.dispatch 为固定唯一的桥接接口，每个方法通过后面的参数来区分，全部为必传参数
  • 第一个参数clazz为类名，String类型
  • 第二个参数method为方法名，String类型
  • 第三个参数json为接口参数，JSON类型，支持嵌套。没有需要传的参数时，传空对象。最外层不支持JSON数组，如需传递
  • 第四个参数为处理结果回调，function类型
    • code Number类型。为0 时表示成功，其他表示失败，code通用错误类型如下表。
    • msg String类型。当error不为0时, 错误描述,方便前端开发排查错误
    • data 全部为JSON类型

# error 通用错误类型

Error 含义
10101 没有找到对应的class
10102 有对应的class，但是没有对应的method
10103 调用class，method对应的接口异常
10201 参数JSON格式异常
10202 参数缺失
10301 权限错误

# 命名规范

采用驼峰命名，第一个参数clazz 首字母为大写，如：User, Router , 其它首字母均为小写。如:

DYBridge.dispatch('Room','changeRoom',{'roomId':'101'},function(){})

  • 接口必要参数缺失时，需要回调对应的error 类型。具体哪个参数缺失，由前端开发排查

# 添加新的桥接接口方法

在对应的类下添加 public static 方法即可，如下：

public static void rn(Context context, Map params, DYBridgeCallback callback) {

}
  • 接口调用为反射调用，方法参数务必保证跟上面一样
  • 每个接口都需要执行 callback 回调给前端结果，同步执行的直接在最后一行加上即可
  • 其它有疑惑的地方可以参考已有桥接接口实现

# 添加新的桥接接口方法

在对应 Module 的 .m 文件中使用 DYBridgeExportMethod 声明对应方法即可。

DYBridgeExportMethod(方法名) {

    // 获取接口传入参数
    DYBridgeParam(参数名称, 预期类型, 是否为必要参数));

    /// 具体实现
    ...

    // 成功回调, data 为返回数据，可以传 nil，但是必须为字典类型
    DYBridgeResolve(data);

    // 异常回调
    DYBridgeReject(错误码, 错误信息);
}
